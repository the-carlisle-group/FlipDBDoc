No rows will be deleted unless all key values exist. No rows will be deleted if there are any outstanding references to any single specified row. In both case errors are signalled.

See also the →[*.Table.MethodList.DeleteWhere] method.

Note that in FlipDB, deletes are logical - no data is destroyed or overwritten. This allows for historical queries, but it also means that deleting rows does not free up disk space. In fact, in FlipDB deleting rows is as expensive as updating rows, and takes up additional space. In certain circumstances, it is useful to physically delete rows, which may be accomplished with the →[ .Table.MethodList.Purge] and →[ .Table.MethodList.PurgeWhere] methods.
